HTTP协议相关学习笔记

  • HTTP是超文本传输协议（英文：HyperText Transfer Protocol，缩写：HTTP）是互联网上应用最为广泛的一种网络协议。是一种client-server协议，也就是说请求通常是由像浏览器这样的接受方发起的通常由像浏览器这样的客户端发出的消息叫做 requests，那么被服务端回应的消息就叫做 responses。
  • HTTP是无状态的，通过HTTP headers让协议变得可扩展。
  • 通常，由HTTP客户端发起一个请求，创建一个到服务器指定端口（默认是80端口）的TCP连接。
  • 发送Ajax的代码：
  • 一个HTTP请求由以下元素组成，一个HTTP method，例如GET，POST；资源路径；HTTP协议版本号；请求头headers；POST这样的方法，报文的body
  • 一个HTTP响应包含以下元素，HTTP版本号；一个状态吗（status code）；一个状态信息；HTTP headers
  • HTTPS就是HTTP over TLS（SSL的前身），HTTP的默认端口是80端口，而HTTPS的默认端口是443。因为HTTP传输过程中有很多的代理和转发，中间的节点都可能篡改信息，因此需要HTTPS。HTTPS的大致工作原理如下，采用RSA非对称加密：
  1. [明文] 客户端发送随机数client_random和支持的加密方式列表
  2. [明文] 服务器返回随机数server_random ，选择的加密方式和服务器证书链
  3. [RSA] 客户端验证服务器证书，使用证书中的公钥加密premaster secret 发送给服务端
  4. 服务端使用私钥解密premaster secret
  5. 两端分别通过client_random，server_random 和premaster secret 生成master secret，用于对称加密后续通信内容
  • HTTP2.0，最新的浏览器都支持，Web服务器支持才能用HTTP2。

压缩头部，客户端和服务端维护一张首部表。如果之前发送过，就不再发送
服务器推送，允许服务端在客户端请求之前主动把内容发送到客户端缓存中，提升性能。
多路复用，可以在一条连接上发送多个请求，也就是同一个域名，开启一个TCP连接，请求都可以在这条连接上干活，因此在 HTTP/2.0 时代，之前的合并 JS、CSS 文件技巧，反而不适用了。
强制SSL
WebSocket，是HTML5标准，保持永久连接，客户端和服务端可以互相发送消息

  • 常见的web安全及防护

sql注入原理
就是通过把SQL命令插入到Web表单递交或输入域名或页面请求的查询字符串，最终达到欺骗服务器执行恶意的SQL命令。
防护：要对用户的输入进行校验，可以通过正则表达式，或限制长度

XSS原理及防范
Xss(cross-site scripting)攻击指的是攻击者往Web页面里插入恶意 html标签或者javascript代码。
防护：对用户输入的地方和变量都需要仔细检查长度和对”<”,”>”,”;”,”’”等字符做过滤；其次任何内容写到页面之前都必须加以encode，避免不小心把html tag 弄出来。

CSRF原理及防护
CSRF就是攻击者盗用了你的身份，以你的名义发送恶意请求，在你登陆了某个网站，生成了Cookie，用户在没有登录A的情况下，访问了危险网站B，而B网站可以用script标签调用A网站的GET请求，这时候Cookie还在，就请求成功了

  • 用户从输入URL到页面加载完成经历了哪些过程？
  1. 浏览器根据请求的URL交给DNS域名解析，找到真实IP，向服务器发起请求；
  2. 进行HTTP协议会话，客户端发送报头(请求报头);
  3. 进入到web服务器上的 Web Server，如 Apache、Tomcat、Node.JS 等服务器;
  4. 进入部署好的后端应用，如 PHP、Java、JavaScript、Python 等，找到对应的请求处理;
  5. 处理结束回馈报头，此处如果浏览器访问过，缓存上有对应资源，会与服务器最后修改时间对比，一致则返回304;
  6. 浏览器开始下载html文档(响应报头，状态码200)，同时使用缓存;
  7. 文档树建立，根据标记请求所需指定MIME类型的文件（比如css、js）,同时设置了cookie;
  8. 页面开始渲染DOM，JS根据DOM API操作DOM,执行事件绑定等，页面显示完成。
  • 讲讲304缓存的原理
  1. 客户端发起 HTTP GET 请求一个文件
  2. 2.服务器处理请求，返回文件内容和一堆Header，当然包括Etag(例如”2e681a-6-5d044840”)(假设服务器支持Etag生成和已经开启了Etag).状态码200
  3. 第二次请求，注意这个时候客户端同时发送一个If-None-Match头，这个头的内容就是第一次请求时服务器返回的Etag：2e681a-6-5d0448402.服务器判断发送过来的Etag和计算出来的Etag匹配，因此If-None-Match为False，不返回200，返回304
    只有get请求会被缓存，post请求不会
